В качестве экспериментов использовал простое консольное приложение на go

Bad practices в CICD файле:
    
1. Запуск пайплайна на pull request на каждую ветку

2. Несколько разных джоб скомбинированы в одну сложную.

3. Игнорирование зафейлиных тестов.

4. Отключение кэша, пайплайн будет работать дольше.

5. Отсутствие Environment, секреты берутся из общих настроек репозитория.

6. Вывод секретов в логи (печатается ***, так как github маскирует)

7. Использование старых версий actions или latest

8. Отсутствие старых версий uses или вообще без указания версии

9. Создание бинарного файла go без флагов, из-за чего он весит значительно больше и содержин ненужную инфу
